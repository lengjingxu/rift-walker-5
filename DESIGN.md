## 一、设计命题

把 "DIABLO BUILD 工具型出装模拟器" 改造为 "裂隙行者 5% —— 一个关于 AI 时代命题的反思型爬塔游戏"。

旧项目累积了丰富的数据基础（6 职业 / 60+ 怪 / 14 传奇 / 10 套 / 5 宝石 / 4 附魔 / 6 配方）但缺游戏性 —— **没有任何玩家决策影响结果**。

新项目核心：

> 玩家在 35 层爬塔中做 *狭义决策*（血瓶投入 / 装备融入 / 战斗技能时机 / 道德选择）。
> 每个决策都改变 *经济资源* 与 *叙事分支*。
> 最终面对的不是「Boss 击败」，而是「我只是一个被测试的 AI」。

## 三、核心循环架构

```
┌─────────────────────────────────────────────────────────────┐
│ [主循环]                                                   │
│  1. 玩家开始新爬塔（选职业 + 选定装备 + 带入血瓶）        │
│  2. 进入第 1 层                                            │
└────────────────┬────────────────────────────────────────────┘
                 ↓
┌─────────────────────────────────────────────────────────────┐
│ [战斗层] 玩家选"继续"后自动跑：                            │
│  ├─ 自动战斗 tick（沿用 simulateBattle 改造为开放循环）   │
│  ├─ 飘字 + 音效（沿用 v1.3）                               │
│  ├─ 资源监控（HP% 触发暂停提示 + 红屏）                    │
│  └─ 击败后跳"决策 modal"                                  │
└────────────────┬────────────────────────────────────────────┘
                 ↓
┌─────────────────────────────────────────────────────────────┐
│ [决策 modal] 三选一：                                      │
│  ▶ 继续冲塔（+层奖励，扣血瓶，保留风险）                  │
│  ↩ 及时撤退（带出当前层奖励及全部已得装备 → 计入排行榜）  │
│  🩸 回城补给（消耗金币买血瓶 / 升级装备 / 融入重练 → 必须  │
│     牺牲当前进度重置爬塔）                                  │
└────────────────┬────────────────────────────────────────────┘
                 ↓
┌─────────────────────────────────────────────────────────────┐
│ [Boss gate] 第 5/10/15/20/25/30/35 层：                    │
│  ├─ Boss 立绘入场（gpt-image 高对比肖像）                  │
│  ├─ 简短叙事幕（200-400 字 boss 悲剧故事）                 │
│  ├─ Boss 战：5/15/25/35 强制开技能决策窗口                │
│  └─ 战后二选一：🩸 终结 / ✋ 不伤（影响分支剧情）          │

                 ↓
┌─────────────────────────────────────────────────────────────┐

│  ├─ 翻转剧情：本以为通关，实际开始"元层"                  │
│  ├─ 视觉切换：游戏风 → 真实照片风（ICU / 植物人 / 脑机接口）│
│  └─ 揭示：玩家是被测试的 AI 意识                           │
└─────────────────────────────────────────────────────────────┘
                 ↓
┌─────────────────────────────────────────────────────────────┐
│ [结局分支] A_Rebel / B_Inheritor / C_Glitched               │
│  └─ 排行榜提交（飞书 Bitable 异步 POST）                   │
└─────────────────────────────────────────────────────────────┘
```

## 六、装备经济系统（融合所有锁值）

### 6.1 词条颜色（Q3 真颜色策略）
| 词条等级 | 色 | 玩家反应 |
|---|---|---|
| T1 黯淡 | ⬜ 白 | 秒卖 |
| T2 褪色 | 🟦 蓝 | 过渡 |
| T3 磨损 | ⬛ 灰 | 升级目标 |
| T4 微光 | 🟧 橙 | 套装基石 |
| T5 闪耀 | 🟨 金 | 保护优先 |
| T6 传说 | 🟪 紫 | **不可卖 / 不可融的奖励品** |

### 6.2 装备稀有度边框
| 稀有 | 边框 | 含义 |
|---|---|---|
| 普通 | 无 | 量产 |
| 魔法 | 蓝边 | 1 条词条 |
| 稀有 | 紫边 | 2-3 条 |
| 传奇 | 金边 | 叙事固定词条 |
| 套装 | 银边 + 套装名 | 触发套装奖励 |

### 6.3 融入（B 曲线·三件共振）
```
3 件同稀有度装 → 融入仪式
├─ 60% → 三件全碎 + 少量金币返回
├─ 35% → 三件消失，生成 1 件高一阶等槽位装
└─ 5% → 三件消失，生成 1 件传奇或 T6 紫词条装
```
**经济崩溃路径**：
1. 玩家贪：5 件全升级 → 碎 2 件 → 资源亏空
2. 玩家赌：融 5 套想出传奇 → 70% 归零 → 无金币买血瓶
3. 第 8 层死：装备全失

### 6.4 升级（线性安全）
```
金币 + 1 件装 → 升级仪式
├─ 80% → 装备 +1 阶（最高 +2 到 T5 变 T4 颜色饱和度更高）
└─ 20% → 装备碎裂 + 金币返还 30%
```
**升级最高只到 T5 → T6 紫词条只能融出**（强制玩家融入）

## 八、两个结局 + 终极翻转（C_Glitched）

### A_Rebel（≥70% 选"不伤"）
- 黑屏 → "0.003 秒 — 你不是最优解，但你是我"
- 回放玩家关键时点 → Q1-Q6 文字浮现

### B_Inheritor（≥70% 选"终结"）
- 黑屏 → "你继承了他，0.3% 是你的笑声"
- Q1-Q6 答案与 A 相反

### C_Glitched（终局后强制 / 由 boss 35 触发）
- **画面风格大切换**：
  - 8-bit 游戏 → **真实照片**（暖黄光 ICU / 床单皱褶 / 真实感拉满）
  - 玩家"化身"为植物人
  - 屏幕外浮现系统文字："测试主体 #LJX-001，攻防演练评估：合格/不合格"
- Q1-Q6 终极揭示："AI 用了 0.003 秒算出了最优解，那个最优解是 *你*"

## 九、排行榜（飞书 Bitable）

**Base**: `LPdqb0BZ3aUjyVsF3Yucb1WFnnc`（"裂隙行者-排行榜"）
**Table**: `tblVBRK5aGowuJsw`（leaderboard）

| 字段 | 类型 | 含义 |
|---|---|---|
| player | text | 玩家昵称 |
| buildHash | text | 当前 build 字符串指纹（30 字符 sha 截断）|
| score | number | 终点层级（35 满分）|
| floor | number | 死亡/撤退层 |
| goldRemaining | number | 死亡时金币 |
| itemsBroughtOut | number | 带出装备数 |
| itemsLost | number | 损失装备数 |
| ending | select | A_Rebel / B_Inheritor / C_Glitched |
| classId | select | paladin / barbarian / sorceress / necromancer / druid / assassin |
| submittedAt | datetime | 提交时间 |

**提交时机**：游戏结束（通关 / 死亡 / 主动撤退）→ 异步 POST，失败本地缓存下次同步。
